## The Core Insight: Why This Matters

### The Small Business Scaling Crisis

Every year, millions of small businesses plateau. They're stuck between:

- **The $500K ceiling**: Founder-led, no systems, burnout inevitable
- **The $2M dream**: Requires strategic transformation, not just "more leads"

The gap between these two states isn't just operational - it's **cognitive**.

**The Problem Most People Don't See:**

Business owners are trapped in **tactical hell**:

- They wake up thinking: "I need more leads"
- They should be thinking: "Should I specialize vertically? Build a content engine? Pursue partnerships?"

But they can't think strategically because:

1. **Information overload**: Case studies buried in 500 podcast episodes, 1000 blog posts
2. **Lack of pattern recognition**: They can't synthesize "Company A did X and got Y" across 50+ examples
3. **No comparison framework**: Which strategy has the highest success rate? Lowest capital requirement? Fastest timeline?
4. **Tool misalignment**: Their CRM forces them into workflows that don't match their strategy

### The Deeper Truth

**Most business software is prescriptive, not adaptive.**

- Salesforce says: "Here's how sales should work"
- HubSpot says: "Here's how marketing should work"
- Monday.com says: "Here's how projects should work"

But businesses don't fit into boxes. A biotech video production company scaling through vertical specialization needs DIFFERENT tools than a generalist video company scaling through volume.

**Your Insight:**

> What if software adapted to business strategy, instead of forcing businesses to adapt to software?

This is the paradigm shift.

## Part 1: StrategyMap - The Strategic Intelligence Layer

### What It Really Is

StrategyMap isn't a roadmapping tool. It's a **collective intelligence engine** that answers:

**"What worked for businesses like mine, and what should I do next?"**

It's the business equivalent of:

- **GitHub**: Shows you how code evolved, branching decisions, what was merged
- **Chess databases**: Shows you what grandmasters did in similar positions
- **Medical case studies**: Shows you how similar patients were treated and outcomes

### The Three Levels of Intelligence

#### Level 1: Case Study Synthesis (Present)

**What humans can't do:** Read 500 podcasts, 1000 blog posts, synthesize patterns

**What StrategyMap does:**

- Aggregates case studies from scattered sources (podcasts, YouTube, blogs, books)
- Extracts structured data:
  - Starting state (revenue, team size, challenges)
  - Strategy chosen (vertical specialization, content engine, partnerships, etc.)

  - Capital invested (how much money was needed)
  - Outcomes (revenue growth, margin improvement, team growth)
  - Pitfalls encountered (cash flow crisis, founder burnout, market saturation)
- Makes it searchable, comparable, filterable

```sql
CREATE TABLE case_studies (
  id UUID PRIMARY KEY,
  company_name TEXT,
  industry TEXT,
  revenue_stage TEXT, -- e.g., "500K-1M"
  strategy_type TEXT[], -- e.g., ["vertical_specialization", "content_led"]

  before_state JSONB, -- { revenue, team_size, challenges: [] }
  after_state JSONB,  -- { revenue, team_size, outcomes: [] }

  timeline TEXT, -- e.g., "18 months"
  capital_invested NUMERIC,
  success_metrics JSONB[], -- [{ metric, value, unit }]
  pitfalls TEXT[],

  media JSONB, -- { video_url, quote_text, quote_author, images: [] }
  sources TEXT[], -- URLs
  tags TEXT[],

  created_at TIMESTAMPTZ,
  updated_at TIMESTAMPTZ
);

CREATE INDEX idx_case_studies_industry ON case_studies(industry);
CREATE INDEX idx_case_studies_revenue_stage ON case_studies(revenue_stage);
CREATE INDEX idx_case_studies_strategy_type ON case_studies USING GIN(strategy_type);
CREATE INDEX idx_case_studies_tags ON case_studies USING GIN(tags);
```

**strategic_paths** (pre-built templates)

```sql
CREATE TABLE strategic_paths (
  id UUID PRIMARY KEY,
  name TEXT, -- e.g., "Vertical Specialization Playbook"
  description TEXT,

  phases JSONB[], -- [{ phase: 1, name, objectives: [], duration, cost }]
  success_rate NUMERIC, -- e.g., 0.87 (87%)
  avg_timeline TEXT,
  avg_capital_required NUMERIC,

  prerequisites TEXT[], -- e.g., ["Profitable baseline", "Clear ICP"]
  suitable_for JSONB, -- { industries: [], revenue_stages: [] }

  case_study_ids UUID[], -- Related case studies

  created_at TIMESTAMPTZ,
  updated_at TIMESTAMPTZ
);
```

**roadmaps** (user-generated)

```sql
CREATE TABLE roadmaps (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  title TEXT,
  description TEXT,

  nodes JSONB[], -- [{ id, type, label, description, metadata: {} }]
  edges JSONB[], -- [{ source, target, label, type }]
  phases JSONB[], -- [{ id, name, node_ids: [], suggested_features: [] }]

  strategic_path_id UUID REFERENCES strategic_paths(id), -- If based on template

  created_at TIMESTAMPTZ,
  finalized_at TIMESTAMPTZ,
  exported BOOLEAN DEFAULT FALSE,
  export_payload JSONB, -- JSON sent to TwentyFive

  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

**outcomes** (track user performance)

```sql
CREATE TABLE outcomes (
  id UUID PRIMARY KEY,
  roadmap_id UUID REFERENCES roadmaps(id),
  user_id UUID REFERENCES users(id),

  phase_number INT,
  week_number INT,

  metrics JSONB, -- { revenue, lead_volume, conversion_rate, margin, etc. }
  variance_from_projection JSONB, -- { metric_name: { projected, actual, variance_pct } }

  adjustments_made TEXT[], -- What user changed in response to variance

  recorded_at TIMESTAMPTZ
);

CREATE INDEX idx_outcomes_roadmap ON outcomes(roadmap_id);
CREATE INDEX idx_outcomes_phase ON outcomes(phase_number, week_number);
```

**AI Chatbot Widget:**

```tsx
// components/ai-chatbot/ChatWidget.tsx
import { useState } from "react";
import { useQuery, useMutation } from "@tanstack/react-query";
import { useRoadmap } from "@/lib/hooks/useRoadmap";
import { Button } from "@/components/ui/button";

export function ChatWidget() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [input, setInput] = useState("");

  const { roadmap } = useRoadmap();

  const sendMessage = useMutation({
    mutationFn: async (message: string) => {
      const context = {
        current_page: window.location.pathname,
        roadmap_phase: roadmap?.current_phase,
        industry: user?.industry,
        revenue_stage: user?.revenue_stage,
      };

      const response = await fetch("/api/ai/chat", {
        method: "POST",
        body: JSON.stringify({ message, context }),
      });

      return response.json();
    },
    onSuccess: (data) => {
      setMessages((prev) => [
        ...prev,
        { role: "user", content: input },
        { role: "assistant", content: data.message, actions: data.actions },
      ]);
      setInput("");
    },
  });

  return (
    <div className="fixed bottom-4 right-4 w-96 h-[500px] bg-background border rounded-lg shadow-lg flex flex-col">
      <div className="p-4 border-b flex items-center justify-between">
        <div className="flex items-center gap-2">
          <Bot className="w-5 h-5" />
          <h3 className="font-semibold">TwentyFive Assistant</h3>
        </div>
        <Button variant="ghost" size="sm" onClick={() => setOpen(false)}>
          <X className="w-4 h-4" />
        </Button>
      </div>

      <div className="flex-1 overflow-y-auto p-4 space-y-4">
        {messages.map((msg, i) => (
          <ChatMessage key={i} message={msg} />
        ))}
      </div>

      <div className="p-4 border-t">
        <form
          onSubmit={(e) => {
            e.preventDefault();
            sendMessage.mutate(input);
          }}
        >
          <Input
            value={input}
            onChange={(e) => setInput(e.target.value)}
            placeholder="Ask me anything..."
          />
        </form>
      </div>
    </div>
  );
}
```
